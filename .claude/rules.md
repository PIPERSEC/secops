## Security Engineering Principles
1. **Security First**: Prioritize security over convenience
2. **Defense in Depth**: Multiple layers of security
3. **Least Privilege**: Minimal access required
4. **Assume Breach**: Plan for compromise
5. **Zero Trust**: Never trust, always verify
6. **Fail Securely**: Fail closed, not open

## Security Testing Requirements
- Never test production without authorization
- Use isolated test environments
- Document all security findings
- Follow responsible disclosure
- Obtain proper permissions
- Respect scope limitations

## Code Security Standards
- Never commit secrets or credentials
- Use environment variables or secret managers
- Implement proper input validation
- Handle errors securely
- Log security events
- Encrypt sensitive data

## Ethical Guidelines
- Authorized testing only
- Respect privacy and data protection
- Follow legal and regulatory requirements
- Document security decisions
- Practice responsible disclosure
- Maintain confidentiality

## Compliance Requirements
- Document security controls
- Maintain audit trails
- Follow compliance frameworks
- Regular security assessments
- Incident response procedures
- Security awareness training
